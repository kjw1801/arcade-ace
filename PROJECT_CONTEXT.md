# PROJECT_CONTEXT.md

> 이 파일은 Claude Code / Claude Cowork 새 세션에서 작업 맥락을 즉시 복원하기 위한 루트 컨텍스트 문서다.
> 작업 폴더(권장: `arcade-senpai/`)의 최상단에 두고, 세션 시작 시 가장 먼저 읽힌다.

---

## 0. 작품 정체

- **제목(노피아 리부트):** 《국정원 에이스는 오락실 관리자를 선배님이라 부른다》
- **구 제목(가제):** 《국정원 S급 관리자》
- **장르:** 현대 첩보 + 착각계 코미디
- **플랫폼:** 노벨피아
- **상태:** 문피아 연재 중단작(50화)을 1화부터 리부트. 기존 50화는 폐기가 아니라
  **원천 자산**으로 사용. 노피아 + 라노벨 핏으로 재설계.
- **원고 위치:** 뮤블 보관, txt 전량 보유.

**제목 자체가 착각 구조의 요약:** 국정원 에이스(강태오)가 오락실 관리자(정우)를
선배로 오해한다 = 작품의 심장인 오독 구조가 제목에 그대로 노출.

---

## 1. 작품 핵심 (절대 유지)

- **주인공 정우:** 본인만 자기가 평범하다고 믿는다. 진짜 능력(오락실/엔지니어 경력)이
  첩보 스킬로 오독되는 **'관성 락' 착각 메커니즘**이 작품의 심장.
- **주요 인물**
  - 강태오 — 오해 점층(국정원 에이스, 정우를 "선배님"이라 부름)
  - 스콜피온 — 외부 관찰 시점
  - D-08 / P-03 — 전투 (P-03은 리듬감 있는 독특한 대사체, 의도적으로 보존)
  - 춘식 — 과묵
  - 유진
  - 유나 — 14살 조카
  - 정하린 — 오독 패턴
- 외부 인물들이 정우를 오독하는 **다중 시점 착각**이 작품의 묘미.

---

## 2. 리부트 확정 결정

- **시점:** 다중 시점 유지. 단 **1화는 정우로 시작**, 정우 분량 비율 압도적.
  외부 오독 시점(스콜피온/강태오)은 **1화당 한 컷, 짧고 강하게** 갭 펀치라인 용도로만.
- **유나(14살 조카):** 유지. 단 **히로인/로맨스 라인 아님** → 코미디 엔진 + 정서 축으로
  기능 재정의. 가족 코미디 톤으로 승부.
- **라노벨 톤:** '살짝' 코팅 — 정우 내적 독백 츳코미, 짧은 호흡, 경쾌한 컷 전환.
  무거운 가족 드라마 톤은 가볍게.
- **노피아 핏:** 1화 **첫 1,000자 안에 갭모에 펀치**. 문피아판의 장례식 회상
  빌드업은 뒤로 빼거나 잘라냄.
- **전략 인지:** 천장(최대 선작 수)보다 **작품 고유색**을 택한 결정. 의도된 트레이드오프.

## 3. 도구

- **awesome-novel-studio** (GitHub: `MJbae/awesome-novel-studio`)
  - Claude Code / Cowork 호환 웹소설 플러그인.
  - 파이프라인: propose → design → create → polish → rewrite
  - 핵심 기능: continuity-bridge(직전 회차 상태 주입) + 16축 polish.
  - 도입 **테스트 단계**. 통째로 갈아타기보다 좋은 부분만 흡수하는 방향.
- **mana-novel 스킬은 이 작품에 사용 금지** — 그것은 《마나를 모르는 외신이 왔다》
  전용 고유명사/설정. 이번 작업의 부산물로 **이 작품 전용 config/design 세트**를 새로 만든다.

## 4. novel-config 설계 방향

```yaml
project:
  title: "국정원 에이스는 오락실 관리자를 선배님이라 부른다"
  target_platform: "노벨피아"
  target_genre: "현대판타지+첩보+착각계 코미디"

guard_rails:
  - "정우는 끝까지 자신이 평범하다고 믿는다 (관성 락 유지)"
  - "외부 인물의 오독은 회수 전까지 해제되지 않는다"
  - "다중 시점 유지하되 정우 시점 분량 최우선"
  - "유나는 로맨스 라인이 아니라 코미디/정서 축으로만 기능"

custom_axes:   # 기본 16축에 추가
  MISREAD:     "오독 상태표 대비 일관성 — 누가 정우를 어디까지 오해 중인지 추적"
  POV_BALANCE: "외부 시점 컷 최소화, 정우 밀착 우선"
  LIGHT:       "라노벨 톤 — 내적 독백 츳코미, 짧은 호흡, 경쾌한 컷 전환 (강도 약)"
  HOOK1K:      "1화 첫 1,000자 내 갭모에 펀치"
```

## 5. 워크플로우

1. **역추출** — 50화 txt에서 추출:
   - 캐릭터 보이스 테이블 (말투·어미·비언어 팔레트)
   - 착각 상태표 (누가 정우를 어디까지 오해 중인지, 몇 화에서 어떻게 심화됐는지)
   - 살릴 장면 목록
2. **재설계** — 위 자료로 `/design-big`을 노피아 타깃으로 재구성 (4번 config 반영)
3. **집필** — `/create EP001`부터 1화 리부트 → `/polish` → 검수 반복

> 50화 전부 없어도 **1~10화만으로 시작 가능** (캐릭터 보이스·착각 메커니즘은 초반에 거의 다 잡힘).

## 6. Codex cross-review (나중 단계)

- ChatGPT Plus + Claude Max 둘 다 구독 중.
- **역할 분담:** Claude Code = 메인(집필·일관성·생산). Codex(GPT-5) = reviewer.
- **Codex에 시킬 것 (메타 피드백만):**
  - 톤 일관성 (정우 톤이 원작 50화와 어긋나는 문장 지적)
  - 첫 1,000자 이탈 위험 구간
  - 외부 시점 컷의 착각 강화 여부 (강화 vs 분량 낭비)
  - 라노벨 코팅 적정성 (과/부족)
- **Codex에 안 시킬 것:** 작품 고유 설정/오독 일관성 (design 문서를 모르므로 Claude polish 담당).
- **도입 시점:** EP001~005를 Claude Code 단독으로 먼저 안정화 → 그때 Codex 비교 →
  polish가 못 잡는 게 나올 때만 상시화. **처음부터 둘 다 돌리지 않는다.**
- **셋업:** OpenAI 공식 Codex 플러그인을 Claude Code에 붙이는 방식이 시작점.
